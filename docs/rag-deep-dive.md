# RAG 技术深度解析

## 1. RAG 的核心思想

RAG (Retrieval Augmented Generation) 的核心思想很简单：**让 LLM 在回答问题时先检索相关知识，再基于检索结果生成答案**。

### 传统 LLM 的问题

1. **知识时效性差**：训练数据有截止日期，无法回答最新问题
2. **缺乏领域知识**：通用模型不了解特定行业的专业知识
3. **幻觉问题**：模型可能会自信地给出错误答案
4. **无法追溯**：无法知道答案的来源

### RAG 如何解决这些问题

```
用户问题 ──▶ 检索系统 ──▶ 获取相关文档 ──▶ 组合提示词 ──▶ LLM 生成 ──▶ 最终答案
                         (找到知识)      (加上上下文)     (基于事实)
```

## 2. RAG 的核心组件

### 2.1 文档加载器 (Document Loader)

负责从各种来源加载文档：

- **文件类**：TextLoader, MarkdownLoader, PyPDFLoader, CSVLoader
- **网络类**：WebBaseLoader, PlaywrightURLLoader
- **数据库类**：NotionLoader, ConfluenceLoader
- **消息类**：SlackLoader, DiscordLoader

### 2.2 文本分割器 (Text Splitter)

为什么需要分割？

- LLM 有上下文长度限制
- 向量检索的精度随文本长度下降
- 分割后可以更精确地定位知识点

常见的分割策略：

1. **固定长度分割**：按字符数分割，简单但可能破坏语义
2. **递归分割**：按段落→句子→单词顺序，更智能
3. **语义分割**：按主题/标题分割，保持完整性

### 2.3 嵌入模型 (Embedding Model)

将文本转换为向量：

- **闭源**：OpenAI text-embedding-3-small/large
- **开源**：sentence-transformers, bge-large-zh

嵌入的核心是**语义相似性**：语义相近的文本，向量距离也近。

### 2.4 向量数据库 (Vector Store)

存储向量并支持相似度检索：

| 数据库 | 特点 | 适用场景 |
|--------|------|----------|
| Chroma | 轻量、易用 | 原型/小规模 |
| Pinecone | 云服务、可扩展 | 生产环境 |
| Weaviate | 开源、GraphQL | 复杂查询 |
| Milvus | 国产、支持混合检索 | 大规模 |

### 2.5 检索器 (Retriever)

从向量数据库中检索相关文档：

- **相似度检索**：返回最相似的 k 个
- **MMR 检索**：平衡相关性和多样性
- **阈值检索**：只返回高于阈值的
- **重排序**：先粗筛再精排

## 3. RAG 的进阶技术

### 3.1 查询优化

- **查询扩展**：将一个问题扩展成多个相关查询
- **查询改写**：将用户口语化的问题改写成检索友好的形式

### 3.2 索引优化

- **父子文档索引**：大文档做父索引，检索子文档
- **摘要索引**：同时索引原文和摘要
- **多向量索引**：一个文档生成多个向量

### 3.3 检索后处理

- **重排序**：使用更精确的模型对结果重排
- **压缩**：去除冗余信息，只保留关键部分
- **过滤**：根据元数据过滤不相关结果

### 3.4 混合检索

结合多种检索方式：

- 向量检索 + 关键词检索
- 密集向量 + 稀疏向量 (BM25)
- 本地检索 + 远程检索

## 4. RAG vs Fine-tuning

| 方面 | RAG | Fine-tuning |
|------|-----|-------------|
| 知识更新 | 快（换文档） | 慢（需重新训练） |
| 成本 | 低 | 高 |
| 可解释性 | 高（可追溯来源） | 低（隐含在模型中） |
| 适用场景 | 动态知识库 | 固定模式/风格 |

## 5. RAG 在 Agent 中的应用

RAG 是 Agent 的重要能力之一：

```
Agent = LLM + 工具 + 记忆 + RAG

Agent 利用 RAG 可以：
1. 查阅文档回答用户问题
2. 基于知识库做决策
3. 实时获取最新信息
4. 减少幻觉，提高准确性
```

典型的 Agent + RAG 工作流：

```
用户: "帮我总结一下这个项目的README"

Agent:
  1. RAG检索 -> 获取 README 内容
  2. LLM总结 -> 生成摘要
  3. 返回结果
```
